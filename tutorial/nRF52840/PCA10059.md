# nRF52840 USB Dongle (PCA10059) 사용 매뉴얼

이 문서는 Mixer를 **nRF52840 USB Dongle (PCA10059)** 에서 빌드·플래시·실행하는 절차를
정리한 매뉴얼입니다. 공식 튜토리얼([README.md](README.md))은 nRF52840 DK (PCA10056)
기준이라 Dongle 사용자에게는 다른 절차가 필요합니다.

핵심 차이점 요약:

| 항목 | nRF52840 DK | nRF52840 Dongle |
|------|-------------|------------------|
| 디버거 | J-Link 내장 | 없음 (DFU 부트로더 사용) |
| 플래시 방법 | `.hex` 파일 USB 마스 스토리지로 복사 | `nrfutil dfu usb-serial` |
| 콘솔 | J-Link CDC 가상 COM 포트 | nRF52840 USBD 직접 (TinyUSB CDC) |
| 노드 ID 입력 | UART 콘솔에서 입력 | 빌드 타임에 hex에 박음 |

## 0. 한 번만 해두는 환경 셋업

이미 설치되어 있으면 건너뛰세요.

### 0.1 SEGGER Embedded Studio (SES) 5.70a

1. https://www.segger.com/downloads/embedded-studio/ 에서 **Older versions →
   Linux x64 5.70a** 를 받아 설치.
2. 설치 위치 확인 — 기본은 `/usr/share/segger_embedded_studio_for_arm_5.70a/`.
   다른 곳에 설치하셨다면 환경 변수 `SES_DIR`로 빌드 스크립트에 알려주세요.
3. 라이선스: nRF 칩에 한해 무료. 첫 실행 시 자동 활성화되거나 `Tools → License
   Manager → Activate Free License (Nordic Semiconductor)`.

> SES 8.x는 새로운 RTL을 쓰는데 Mixer의 stdio.c가 옛 RTL API에 의존하므로 빌드가
> 깨집니다. 반드시 5.x 계열 (5.24, 5.70a 등) 을 쓰세요.

### 0.2 nrfutil

DFU로 펌웨어를 굽기 위한 Nordic 공식 도구입니다.

```bash
mkdir -p ~/.local/bin
curl -L https://files.nordicsemi.com/artifactory/swtools/external/nrfutil/executables/x86_64-unknown-linux-gnu/nrfutil \
    -o ~/.local/bin/nrfutil
chmod +x ~/.local/bin/nrfutil
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
export PATH="$HOME/.local/bin:$PATH"

nrfutil install nrf5sdk-tools
nrfutil install device
nrfutil --version  # 확인
```

### 0.3 시리얼 권한

`/dev/ttyACM*`를 sudo 없이 쓰려면 `dialout` 그룹에 속해야 합니다:

```bash
groups | grep -q dialout || sudo usermod -aG dialout "$USER"
# 위 명령을 처음 실행했다면 로그아웃/재로그인 필요
```

## 1. 매번 새 동글에 적용하는 워크플로

가장 자주 하게 될 작업입니다. 동글 N개를 처음 받았다고 가정하고 ID=1, 2, ... N을
구워 동작시키는 흐름입니다.

### 1.1 노드 구성 확인

[mixer_config.h](mixer_config.h) 의 `nodes[]`, `payload_distribution[]` 가 사용할
노드 ID 집합과 일치하는지 확인하세요. 예를 들어 4노드로 늘리려면:

```c
static const uint8_t nodes[] = { 1, 2, 3, 4 };
static const uint8_t payload_distribution[] = { 1, 2, 3, 4, 1, 2, 3, 4 };
```

`MX_NUM_NODES`, `MX_GENERATION_SIZE` 같은 매크로는 자동으로 따라옵니다.

### 1.2 각 노드 ID에 대해 펌웨어 빌드

레포 루트에서:

```bash
scripts/build_node.sh 1     # ID=1 펌웨어 → tutorial/nRF52840/build/node1/
scripts/build_node.sh 2     # ID=2 펌웨어 → tutorial/nRF52840/build/node2/
# ... 필요한 만큼 반복
```

스크립트가 하는 일:
- `tutorial/nRF52840/node_id.h` 생성 (`#define BUILD_NODE_ID N`)
- SES emBuild로 PCA10059_Release 구성 빌드
- ELF에서 TOS_NODE_ID가 정확히 N으로 박혔는지 검증
- nrfutil로 DFU 패키지(`mixer_node{N}.zip`) 생성

각 빌드 산출물은 `tutorial/nRF52840/build/node{N}/` 에 격리되어 들어갑니다.

### 1.3 동글에 굽기

각 동글에 대해:

1. 동글을 USB 포트에 꽂기
2. **RESET 버튼을 한 번 짧게 눌러 부트로더 모드 진입** — 빨간 LED가 빠르게 깜빡이면
   진입 성공
3. 굽기 (자동 감지):
   ```bash
   scripts/flash_node.sh 1   # ID=1 펌웨어를 DFU 모드인 동글에 굽기
   ```
   여러 동글이 동시에 DFU 모드면 스크립트가 거부하니, 그땐 명시:
   ```bash
   scripts/flash_node.sh 1 /dev/ttyACM2
   ```
4. 굽기 끝나면 동글이 자동으로 펌웨어 모드로 부팅.

### 1.4 동작 확인

모든 동글이 USB에 꽂혀 있으면 `Mixer Console (PCA10059)` 로 인식됩니다:

```bash
lsusb | grep 1915
# Bus 001 Device XXX: ID 1915:000a Nordic Semiconductor ASA Mixer Console (PCA10059)
# (동글 수만큼 줄이 나와야 정상)

ls /dev/ttyACM*
# 각 동글마다 하나씩 ttyACM* 가 생김
```

LED 패턴:
- **빨강 (P0.08)**: TX — 패킷 송신 시 깜빡임
- **파랑 (P0.12)**: RX — 패킷 수신 시 깜빡임
- 두 노드 이상이 라운드를 같이 돌면 두 색이 거의 동시에 깜빡임

시리얼 콘솔로 라운드 통계 확인:

```bash
# screen, minicom, picocom 등 아무거나
screen /dev/ttyACM1 115200
# 또는
picocom -b 115200 /dev/ttyACM1
```

기대 출력:

```
preparing round 100 ...
starting round 100 ...
statistics:
num_sent: 8
num_received: 5
slot_decoded: 11
slot_off: 13
radio_on_time: 13227us
# ID:1 round=100 rank=8 dec=8 !dec=0 weak=0 wrong=0
```

`rank=8 dec=8 !dec=0` 이면 모든 메시지가 정상 디코드된 것 — 이상적인 동작입니다.

## 2. 트러블슈팅

실제로 만나본 문제들과 대처법.

### 2.1 굽고 나서 USB가 인식이 안 됨 (LED는 정상)

- **증상**: `nrfutil dfu` 가 "Device programmed." 출력 후, `lsusb`에 `1915:000a` 가
  안 나타나거나 `/dev/ttyACM*` 가 잡히지 않음. 그러나 LED(빨강)는 깜빡임.
- **원인**: 호스트 측 USB 포트 상태가 stale. 부트로더 → DFU → 펌웨어 재부팅 시
  연속된 enumeration을 호스트가 처리 못한 경우.
- **해결**: 동글을 **다른 USB 포트**로 옮기면 즉시 해결됩니다. 같은 포트로 다시
  꽂아도 됩니다. 펌웨어 자체는 정상 동작 중이므로 굽기를 다시 할 필요는 없습니다.

### 2.2 `Could not resolve host: github.com` 같은 DNS 에러 (push 시)

- **원인**: 사내 DNS 서버가 응답이 느려서 짧은 timeout으로 실패.
- **해결**: 일시적으로 공용 DNS 사용:
  ```bash
  sudo resolvectl dns <interface> 1.1.1.1 8.8.8.8
  ```
  `<interface>`는 `ip route show default | awk '{print $5}'` 로 확인.

### 2.3 LED 모두 꺼짐 (펌웨어가 부팅 안 함)

- **증상**: 굽기 직후 LED가 전혀 안 깜빡임, 다른 USB 포트로 옮겨도 마찬가지.
- **원인 후보**:
  1. 메모리 맵이 부트로더 영역(0xE0000+)을 침범 → 매우 드묾, 펌웨어 크기가 약
     800KB를 넘을 때만 발생. 일반 Mixer 펌웨어는 80KB 정도라 무관.
  2. 빌드된 hex의 시작 주소가 0x1000이 아닌 0x0 → memory map XML이 깨졌을 때만
     발생. 정상이라면 다음 명령으로 0x1000으로 확인 가능:
     ```bash
     head -1 tutorial/nRF52840/build/node1/Tutorial_node1.hex
     # ":10 1000 00 ..." 으로 시작해야 함 (1000 = 시작 주소)
     ```
- **복구**: RESET 버튼 짧게 눌러 부트로더로 복귀하면(빨강 깜빡임) 다른 펌웨어로
  다시 구울 수 있습니다. 부트로더 자체는 절대 안 망가집니다 (메모리 맵이
  보존하므로).

### 2.4 RESET을 눌러도 부트로더로 안 들어감

- **원인**: RESET 누르는 시간이 너무 길거나, 동글의 RESET 핀이 다른 신호와 연결.
- **해결**: 빠르게(0.1초 정도) 한 번만 누르세요. 길게 누르면 그냥 리셋됩니다.
- **주의**: 부트로더 자체는 펌웨어가 아무리 망가져도 살아있도록 설계되어 있습니다.
  RESET이 정말 안 되면 USB를 뺐다 꽂으면서 RESET을 누르는 방법도 있습니다.

### 2.5 빌드 실패 — "unknown type name '__printf_tag_ptr'"

- **원인**: SES 8.x 사용 중. Mixer의 stdio.c는 옛 RTL API를 씁니다.
- **해결**: SES 5.x 계열로 다운그레이드. 0.1 절 참고.

### 2.6 빌드 실패 — `emBuild not found at ...`

- **원인**: SES 설치 위치가 스크립트 기본값과 다름.
- **해결**: 환경 변수로 알려주기:
  ```bash
  SES_DIR=/path/to/segger_embedded_studio_for_arm_5.70a scripts/build_node.sh 1
  ```

### 노드 ID 주입 방식

스크립트가 `tutorial/nRF52840/node_id.h` 를 임시 생성합니다:

```c
#define BUILD_NODE_ID 3
```

main.c가 `__has_include("node_id.h")` 로 자동 픽업해서 `TOS_NODE_ID` 초기값으로 씁니다.
`node_id.h` 가 없으면 매크로가 0으로 폴백 → 기존 UICR 입력 경로가 활성화됩니다 (DK
호환성 유지).

### USB CDC와 Mixer 타이밍

USB IRQ는 NVIC priority 7(가장 낮음) 로 설정되어 라디오/타이머 ISR을 절대 preempt
하지 않습니다. `tud_task()` 호출도 main.c의 라운드 시작 대기 루프 안에서만 일어나고
라운드 진행 중에는 호출되지 않습니다 — 이로써 USB가 슬롯 동기화를 깨뜨릴 가능성을
원천 차단합니다.

검증된 결과: 두 노드 동작 시 매 라운드 `rank=8 dec=8 !dec=0` 달성 (모든 메시지 100%
디코드).
